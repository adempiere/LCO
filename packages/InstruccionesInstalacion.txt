AUTOR: Carlos Antonio Ruiz Gómez - globalqss
COMPAÑÍA: Quality Systems & Solutions QSS Ltda.


Para versiones de Adempiere >=  3.4

- Instale adempiere normalmente
Hay cantidades de guías de instalación en el wiki:
http://adempiere.com/wiki/index.php/ADempiere_Installing

- Una vez instalado puede crear una nueva empresa utilizando el PUC colombiano provisto para este fin
http://adempiere.com/wiki/images/a/a5/AccountingCO.zip
no olvide descomprimirlo y seleccionar el csv resultante cuando esté creando un nuevo cliente


Para instalar la localización Colombia:

- Descargue el archivo packages_LCO340.jar
http://adempiere.svn.sourceforge.net/viewvc/*checkout*/adempiere/contributions/Localizations/Colombia/packages/packages_LCO340.jar

- Cree el directorio $ADEMPIERE_HOME/packages/LCO/lib
en windows -> %ADEMPIERE_HOME%\packages\LCO\lib

- Copie en este directorio el archivo packages_LCO340.jar

- Ejecute RUN_setup.[bat/sh] o RUN_silentsetup.[bat/sh]
este paso debe integrar en el Adempiere.jar las clases de la localización Colombia

- Descargue el paquete 2pack LCO_Retenciones.zip
http://adempiere.svn.sourceforge.net/viewvc/*checkout*/adempiere/contributions/Localizations/Colombia/packages/LCO_Retenciones.zip

- Ingrese en Adempiere y ejecute el programa PackIn con los siguientes valores:
Type: File
Source: [directorio_donde_descargo]/LCO_Retenciones.zip
Package dir: c:\temp   -- cualquiera con permiso de escritura, es temporal
NO MARCAR Override Dict


OJO!! Importar el package LCO_Retenciones.zip - adiciona un model validator para todas las empresas existentes al momento de importar.

Si desea usar LCO en una empresa instalada después de importar el package, debe ingresar como el administrador de esa empresa (ej: GardenAdmin) abrir la ventana Client (Compañía)
http://adempiere.com/wiki/index.php/ManPageW_Client
y llenar el campo "Model Validator" (Clase de Validación del Modelo) con:
org.adempiere.model.LCO_Validator
Si ese campo ya tiene alguna clase puede añadir la nueva separada por punto y coma
Ej en GardenWorld:
compiere.model.MyValidator;org.adempiere.model.LCO_Validator

Si no puede ingresar a la compañía porque Adempiere saca error indicando que falta la clase model validator, puede ingresar como Administrador del Sistema (System), ingresar a la ventana System
http://adempiere.com/wiki/index.php/ManPageW_System
y deshabilitar la opción "Fail on missing model validator".
NO ES RECOMENDABLE DEJAR ESTA OPCION DESHABILITADA - es solamente para poder ingresar como el usuario de la compañía y configurar el model validator.  Después de esto se recomienda volver a dejar la opción habilitada.


---------------------------

Despues de esto puede importar también el package LCO_RetencionesBaseDataGardenWorld.zip
http://adempiere.svn.sourceforge.net/viewvc/*checkout*/adempiere/contributions/Localizations/Colombia/packages/LCO_RetencionesBaseDataGardenWorld.zip

Este importa los datos básicos de tipos de contribuyente, CIIU, tipos de retención, etc
---> EN GARDENWORLD

Desafortunadamente no hay en este momento como indicar a 2pack que cargue un archivo en un cliente diferente.  Si se desea cargar los datos en una empresa diferente debe descomprimir el archivo LCO_RetencionesBaseDataGardenWorld.zip, editar el archivo 
LCO_RetencionesBaseData\dict\PackOut.xml
y cambiar las ocurrencias de GardenWorld por un valor apropiado dependiendo de su instalación.

---------------------------

NOTA IMPORTANTE:
Después de importar paquetes con 2pack es recomendable (algunas veces necesario) correr cuatro procesos:

1 - En la ventana Lenguaje (Language) seleccionar los lenguaje del sistema (debe ser es_CO si siguió las instrucciones aquí)
Cuando tenga el registro del lenguaje seleccionado en pantalla oprimir el botón "Mantenimiento del lenguaje" (Language Maintenance)
Y seleccionar "Adicionar traducciones faltantes" (Add missing translations)

2 - Proceso "Sincronizar Terminología" (Synchronize Terminology)

3 - Proceso "Comprobar Secuencia" (Sequence Check)

4 - Proceso "Actualización de Rol" (Role Access Update) - tener en cuenta que se debe dejar el rol en blanco para que los tome todos.
